# 投放数据平台 — 运行说明

## 目录结构
```
china-marketing-board\
├── 启动平台.bat        ← 双击启动后端服务(可见控制台,实时日志)
├── 手动回填.bat        ← 双击手动跑一次全量历史回填(断点续跑)
├── pipeline\           ← 全部后端代码
│   ├── app.py          FastAPI 服务(接口 + 定时调度 + 前端托管)
│   ├── fetchers.py     四平台各维度抓取器
│   ├── crawl.py        滚动抓取(近N天)
│   ├── backfill.py     历史回填(断点续跑)
│   ├── auth_login.py / multi_login.py   浏览器登录取 token
│   ├── db.py           入库(UPSERT)
│   ├── creds.json      (已迁移到数据库 accounts 表, 仅作备份)
│   ├── profiles\       各登录的浏览器配置(记住登录态)
│   └── web\            前端源码(Vue), web\dist 是构建产物
└── 数据库: PostgreSQL  ad_data  (表 ad_daily / accounts / tasks / runs / crawl_progress)
```

## 怎么用

### 1. 启动平台(每次开机后)
双击 **启动平台.bat** → 弹出黑窗口显示实时日志，服务起在 8770 端口。
- 本机看板/后台:  http://localhost:8770
- 团队(内网)访问:  http://10.102.130.181:8770

只要这个黑窗口开着，服务就在跑（含每5分钟的滚动抓取）。关掉窗口=停服务。

### 2. 看数据 / 管账号 / 管任务
浏览器打开上面的地址：
- **数据看板**：按平台/账号/层级/指标/日期看趋势
- **账号管理**：看各登录的登录态；过期时点「刷新登录」会弹浏览器让你重登
- **定时任务**：看滚动任务状态和运行记录

### 3. 补历史 / 手动回填
双击 **手动回填.bat**（可反复运行，断点续跑，已抓的跳过）。

## 常见问题
- **看不到窗口/黑框**：说明服务没在前台跑。双击 启动平台.bat 即可。
- **某账号登录态 expired**：账号管理页点「刷新登录」，在弹出的浏览器里重登。
- **换机器/IP变了**：内网地址里的 10.102.130.181 换成新机器的内网 IP。
- **开机自启**：可把 启动平台.bat 加进 Windows 任务计划程序(开机触发)。
