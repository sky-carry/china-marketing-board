## 1. 平台架构（心里有数）

两个 Docker 容器，配置在 `docker-compose.yml`：

| 容器 | 作用 | 端口 |
|---|---|---|
| `china-marketing-board-app-1` | FastAPI 后端 + 前端页面 + 定时抓取 | 对外 **8060** |
| `china-marketing-board-db-1` | PostgreSQL 18 数据库(ad_data) | 仅本机 5433 |

- 数据存在 Docker 命名卷 `cmb_pgdata`（删容器不丢数据）。
- 抓取的登录凭证存在库的 `accounts` 表；广告数据存 `ad_daily` 表。

## 3. 更新代码后重新部署

> 平时改了代码、想让线上生效，走这套。**数据不受影响**。

方式 A（推荐，从 GitHub 拉）——首次需在服务器配好 git 拉取权限：
```bash
cd /root/china-marketing-board
git pull
docker compose up -d --build app     # 只重建应用，几分钟；db 不动
```

方式 B（本地打包上传，无需服务器 git 权限）：在本地仓库执行
```bash
# 本地
git archive --format=tar.gz HEAD -o code.tar.gz
# 用任意工具把 code.tar.gz 传到 /root/china-marketing-board/
# 服务器
cd /root/china-marketing-board && tar xzf code.tar.gz
docker compose up -d --build app
```

> 说明：前端改动会在构建时自动重新打包（多阶段 Dockerfile）。
> 构建慢属正常（装依赖/浏览器），已配国内镜像。构建期间旧容器仍在服务，完成才切换。

## 4. Token 刷新（登录凭证保活）

抓数据靠各平台的登录 token，会过期。分两类：

### 沸点 / 微橙 / 麦斯 —— 全自动，无需管
- 应用每 3 小时自动保活；抓取失败也会用库里存的账号密码自动重登。
- 想手动刷某个账号：
  ```bash
  docker compose exec app python auth_autologin.py <账号ID>
  ```

### 小飞机 —— 需短信验证码，手动（在本地电脑操作）
服务器没有可视浏览器、也收不到短信，所以小飞机登录要在**你自己的电脑**上做：

1. 双击本地仓库根目录的 **`小飞机登录.bat`**。
2. 输入账号ID（`1`=sdjr / `5`=ayh / `6`=135796；也可在"账号管理"页看）。
3. 弹出的"隧道"窗口里输入服务器 root 密码，保持窗口开着。
4. 自动打开浏览器登录小飞机，完成短信验证。成功后凭证会**直接写进服务器库**。
5. 完事关掉隧道窗口。

> 何时需要做：账号管理页某个小飞机账号显示"需手动登录/已过期"，或看板该账号没数据时。

## 5. 定时抓取

应用启动后自带调度（无需手动开）：

- **每 5 分钟**：滚动抓当日数据。
- **每日 08:30**：补抓近 15 天（防止偶发漏抓）。
- 用 Postgres 建议锁保证只有一个实例在跑，不会重复。

在页面"定时任务"里可看运行记录、手动触发。

## 6. 数据库备份 / 查询

备份（导出到文件）：
```bash
cd /root/china-marketing-board
docker compose exec -T db pg_dump -U postgres ad_data | gzip > backup_$(date +%F).sql.gz
```

进数据库查数据：
```bash
docker compose exec db psql -U postgres -d ad_data
# 例：各平台行数
#   SELECT platform, count(*) FROM ad_daily GROUP BY platform;
```

恢复（谨慎，会覆盖）：
```bash
gunzip -c backup_xxxx.sql.gz | docker compose exec -T db psql -U postgres -d ad_data
```

## 7. 常见问题排查

| 现象 | 排查 |
|---|---|
| 网页打不开 | `docker compose ps` 看 app 是否 Up；`docker compose logs --tail 50 app` 看报错 |
| 登录提示密码错 | 默认 skg/skg@A168；忘了可进库重置(见下) |
| 某账号无数据 | 多半 token 过期：沸点/微橙/麦斯 等自动重登或手动 `auth_autologin.py <id>`；小飞机走 `小飞机登录.bat` |
| 更新后被登出 | 正常范围内；用默认账号或你改后的密码重新登录（已做密钥持久化，一般不会） |
| 内存告警 | `free -h`；已配 2G swap，Chromium 重登时短暂升高属正常 |

重置登录密码（忘记时，进库直接删账号让其重建默认）：
```bash
docker compose exec db psql -U postgres -d ad_data -c "DELETE FROM auth_users;"
docker compose restart app        # 重启后自动重建默认 skg/skg@A168
```
