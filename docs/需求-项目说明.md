# 需求：SKG 效果广告投放数据聚合平台

## 一、为什么做这个项目

SKG（健康电器品牌）的效果广告，投在**多家第三方投放工具**上（小飞机、沸点、微橙、麦斯等）。问题是：

- **数据分散**：每家投放工具一个后台，账号密码各一套，数据各存各的；
- **口径不一**：各家报表的指标名称、字段含义、金额单位都不一样，没法直接放一起比；
- **靠人工汇总**：每天要挨个后台登录、导出报表、粘到 Excel 里手工对齐，**费时、易错、不及时**；
- **看不全**：花了多少、带来多少订单、ROI 多少、哪个账户/计划好，散在各处，管理层看不到全局。

一句话：**多平台投放的数据太散，人工整合又慢又容易错，需要一个自动汇总、口径统一、随时能看的看板。**

## 二、这个项目做什么

一句话概括：**定时自动抓取各家投放后台的数据 → 归一成统一指标 → 存进数据库 → 用看板展示。**

## 三、核心需求

| # | 需求 | 说明 |
|---|---|---|
| 1 | 自动抓取 | 定时登录各投放平台后台，自动拉取投放数据，无需人工导出 |
| 2 | 口径统一 | 各家不同的字段/单位归一成统一指标（金额统一到元、时间统一时区等），能横向对比 |
| 3 | 数据看板 | 按平台/账户/计划等维度看花费、订单、ROI 等核心指标，支持按日期、多条件筛选 |
| 4 | 订单明细 | 逐笔电商订单可查（含归因、回传状态等），支撑归因分析与回传核对 |
| 5 | 账号与凭证管理 | 集中管理各平台登录账号，凭证过期能自动/半自动重登，抓取不中断 |
| 6 | 稳定运行 | 定时任务后台常驻，抓取失败能重试，限流能退避，尽量不丢数据 |

## 四、价值

- **省人力**：告别每天挨个后台导表、手工对齐 Excel；
- **看得全、看得快**：全平台投放效果一屏看清，接近实时；
- **决策有据**：ROI、归因、订单等口径统一，能真正拿来做投放优化决策。

## 五、系统组成（简述）

- **抓取层**：各平台各维度的抓取器，逆向后台接口，归一化后入库；
- **数据库**：PostgreSQL，统一存储投放数据与订单明细；
- **服务/看板**：FastAPI 后端 + Vue 看板，含查询、账号管理、定时任务、订单明细等页面；
- **部署**：本地 + 云服务器双实例，各跑一套完整系统。
