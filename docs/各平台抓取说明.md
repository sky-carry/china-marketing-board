# 各平台抓取说明

> 每个平台"爬什么、怎么爬、字段怎么归一"的权威说明。代码实现见 [`pipeline/fetchers.py`](../pipeline/fetchers.py)（源头真相），本文是人类可读版。
> 统一抓取入口：`fetch(login, level, day)` → 返回归一化后的行列表，再 UPSERT 进 `ad_daily`。**只返回当天有消耗(cost>0)的行**。

---

## 零、通用机制

### 统一入库口径（ad_daily 的 DBCOLS）
所有平台抓回来的原始字段，都在各自 fetcher 里抹平差异、映射到同一套字段：

- 主键：`platform, login_account, level, entity_id, date`
- 基础：`entity_name, account_id, account_name, parent_id, parent_name, channel`
- 指标：`cost 消耗 / impressions 展示 / clicks 点击 / ctr / cpm / cpc / conversions 转化 / conversion_cost`
- 成交：`orders 下单量 / pay_amount 下单金额 / roi`（gross）、`real_orders / real_pay_amount / real_roi`（net，去水分/成交口径）、`refund_rate 退款率`
- 直投归因：`direct_orders/pay_amount/roi`（直投下单 gross）、`direct_real_orders/pay_amount/roi`（直投成交 net）

### 抓取与调度
- **并发**：`crawl_window` 以「(登录, 层级)」为工作单元线程池并发，单元内按天串行；每单元独立 DB 连接。
- **定时**：当日每 5 分钟实时 + 近 15 天每日 08:30 补数；多实例用 Postgres 建议锁防重复跑。
- **重试**：网络瞬断（SSL/超时/连接重置/HTTP 429·5xx）自动重试 2 次；auth 失效立即中断并触发重登。
- **去重**：`crawl_progress` 记录已抓组合幂等跳过；每轮抓取后自动执行「平台迁移去重」（见文末）。
- **历史账号**：`accounts.is_historical=true` 的不参与定时抓取（如方块、停投的小飞机账号），数据保留。

### 登录态维护
- **纯账密自动登录**（沸点/微橙/麦斯/博擎/小飞机常规）：`auth_autologin.py`，无头浏览器填账密，后台每 3 小时保活、抓取失败自动重登。
- **需人工**：小飞机会话彻底失效需短信验证码 → 标 `need_login`，本地 `小飞机登录.bat`（SSH 隧道到服务器库，本机开浏览器登录写回）。
- **方块**：图形验证码（算术题），无法全自动，靠一次性登录拿 token。

## 一、小飞机（买量小飞机）

| 项 | 说明 |
|---|---|
| 站点 | `td.smallfighter.com`（广告管理/报表）、`cid.smallfighter.com`（CID 千川电商/素材报表） |
| 媒体 | 巨量引擎（抖音/头条）+ 千川 CID 电商 |
| 登录 | `auth={token, op_uid/op_uids, sid}`（Bearer + cookie `td.sid`/`td-op-uid`）；账密自动登（填邮箱/密码回车）；CID 另抓 `cid_token`。会话彻底失效需短信验证码人工登。 |
| 多项目 | 一个登录可挂多个「项目」，`op_uid` 即项目 ID；`op_uids` 有值则逐项目抓 |
| 维度 | 推广账号 / 广告组 / 广告 / 创意 / 素材 |

**接口**
- 广告组/广告/创意：`POST td.smallfighter.com/v1/{plan,campaign,creative}/list`，body 含 `metrics=XFJ_MET`、`_type`、`begindate/enddate`（unix 秒，北京时间当天 0 点 = UTC-8）。
- **推广账号（账户级）**：走 `POST cid.smallfighter.com/v1/cid/report/list`（`Keys=["account_id"]` 聚合成一账户一行，`_type=CID_REPORT`）。⚠️ **坑**：`v1/account/list` 只含「广告管理」里的账户、会漏掉停投账户；CID 报表是超集（含停投），所以账户级改走 CID。
- 素材：CID 系统独立接口。

**字段/缩放/映射**
- 缩放：金额 ÷1e6（如 cost、cid_trans_payment_amount）；比率（ctr、refund）÷100。
- `entity_id`：**账户级 = `ExternalId`（巨量广告主 ID，长号）**；广告/广告组 = `campaign_id`/`plan_id`（小飞机内部短号）；`account_id` 字段统一取巨量长号（`AccountExternalId`）。
- 直投=「直推」：`direct_orders=adx_metric_323`、`direct_pay_amount=adx_metric_326/1e6`、`direct_roi` 按 付款/消耗 算；`direct_real_*=cid_trans_direct_real_*`。

## 二、沸点（沸点投放）

| 项 | 说明 |
|---|---|
| 站点 | `admin.fifay.cn`（网页）、`api.fifay.cn`（接口） |
| 媒体 | 抖音生态 |
| 登录 | **官方开放 API：`Authorization: Bearer <api_key>`（稳定，平台发放）**。旧法（备份 `fetch_fd_legacy`）：网页登录态 `token`+`did` 头。入口 `fetch_fd` 按 `auth.api_key` 有无自动切换 |
| 维度 | 账户维度 / 项目维度 / 计划维度 / 素材维度（+ 订单明细） |

**接口**
- 报表：`POST api.fifay.cn/fifay-ad/report/union/get`，body=`{adReportType, startDate, endDate(同日), current, pageSize, customizeFields}`。
  - ⚠️ **坑**：`direct*`/间接字段**必须在 `customizeFields` 里显式请求**才返回，否则为 0。
- 订单：`POST report/order/get`（订单明细：`orderNo/goodsInfo/payMoney/adName/advertiserName/tracePoint` 等）。

**字段/缩放/映射**
- 缩放：金额 ÷100（分→元）；ROI ÷100（百分比→倍）；比率 ÷100（得百分数值）。
- `entity_id`：`advertiserId`（巨量广告主 ID）。
- 直投=「直接」：gross=`directOriginOrderCount/Amount/Roi`，net=`directOrderCount/Amount/Roi`。
- 总/真实：`orders=originOrderCount`（原始），`real_*=orderCount/Amount/Roi`（有效）。

| 维度 | 账户（逐日：`startDate=endDate=同一天`） |

**接口**：`POST /prod-api/superoceanengine/newSuperOceanengineAdvertisers/selectSuperOceanengineAdvertisersAllData`，body=`{pageNum, pageSize, differentiateLineType:"2", lineType:"", selectedField:"", advertiserId:null, startDate, endDate}`。

- 退款率=`goodsPriceRefundRate`（订单金额退款率）。

## 七、阿里妈妈 UD 智汇投（UDS）

| 项 | 说明 |
|---|---|
| 站点 | `ud.alimama.com`（UD 智汇投 udSmart） |
| 媒体/成交 | 投放巨量引擎（`directMediaId=tou_tiao`），成交回流淘系（天猫/淘宝，`alipay*`=支付宝成交） |
| 登录 | **登录 cookie**（`accounts.auth.cookie`）+ `csrfId` + `dynamicToken`（存 auth，服务器不严格校验、过期值也能用）。**阿里/淘宝登录强反爬（滑块+`navigator.webdriver`检测），Playwright 无法自动登录** → cookie 需**人工定期刷**（类似小飞机需人工），不在 `PW_PLATFORMS` |
| 维度 | 账户（`level=账户`；一次拿全部账户，逐账户一行） |

**接口**：`POST advertiser/horizontal/findPage.json`（form 表单）。关键 body：`bizCode=udSmart`、`directMediaId=tou_tiao`、`unifyTimeType=alipayTime`、`orderField=charge&orderBy=desc`、`pageSize=100`、`rptQuery={fields:"charge,afterRefund...",conditionList:[{sourceList:["scene","advertiser_list"],effectEqual:"15",unifyType:"last_click_sync_async",startTime,endTime,isRt}]}`。
- **日期**：`rptQuery.conditionList[0].startTime/endTime`（同一天=单日）；`isRt`：当天=`true`（实时），历史=`false`。`effectEqual=15`=归因周期15天。
- ⚠️ **坑**：账户列表接口返回的账户对象顶层**没有指标**，**每个账户的指标在 `reportInfoList[0]` 里**（要设了 `rptQuery` + `isRt`/日期才有；只看顶层会以为没数据）。`report/query.json` 只给「整体按天」汇总，拆不到账户，别用。

**字段/缩放/映射**（对照飞书《字段对应表》UDS 列，金额=元、ROI 自然值，**无缩放**）
- `entity_id`=`directAdvertiserId`（即巨量广告主ID，与小飞机/沸点同源）；`entity_name`=`name`。
- 消耗=`charge`；展现=`adPv`、点击=`click`、`ctr`/`ecpm`/`ecpc`；转化=`convertNums`/`convertCost`。
- 总（下单）：`orders=alipayInshopNum`、`pay_amount=alipayInshopAmt`（总成交金额，**非** `gmvInshopAmt`）、`roi`。
- 真实（剔除退款）：`real_pay_amount=afterRefundAlipayAmt`、`real_orders=afterRefundAlipayNum`、`real_roi=afterRefundRoi`；退款率=`refundRate`。
- 直投=「直接成交」：gross=`alipayDirNum/alipayDirAmt/dirRoi`，net(剔除退款)=`afterRefundAlipayDirNum/Amt/Roi`。
- 与其他平台一致：消费0但有成交/退后/单品/订单的行也保留（付款回流到无消费的天）。

## 八、通用数据规范

### 平台迁移去重
投放机构把账户从一个平台迁到另一个（已知：**小飞机 → 沸点**）后，迁出方会残留同账户同日的**不完整数据**（通常只有消费）。规则：同 `账户ID + 日期` 同时出现在迁出方账户层和迁入方账户层时，**删迁出方、留迁入方**。逻辑在 `db.dedupe_migrated_accounts()`（配置 `MIGRATION_DEDUP`），每次抓取后自动执行。详见 [业务说明.md](业务说明.md#七数据规范平台迁移去重)。

### 接新平台的通用方法（以方块为例）
1. **登录**：抓包看登录接口；账密直登最好，验证码则想办法（方块的算术验证码是读图算答案）。
2. **找报表接口**：登录后在报表页抓 XHR / 或扒前端 JS 找接口路径与参数；注意日期参数名、分页、"自定义列"是否影响返回字段。
3. **确认粒度**：单日查询能否返回按天数据（ad_daily 是按天的）。
4. **核对字段映射**：把平台字段对到统一口径，确认缩放规则、real/direct 用哪个口径（拿不准问需求方）。
5. **落地**：写 `fetch_xx` + 注册到 `FETCH`/`LEVELS`；一次性历史回填或接入定时。
