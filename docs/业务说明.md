# 四平台投放数据平台 · 业务说明

> 给不熟悉投放业务的同学快速上手：这套系统是做什么的、SKG 是谁、四个平台各干嘛、每个数据字段什么意思。

---

## 一、这套系统属于什么业务模式

一句话：**效果广告投放（电商买量）的数据聚合平台**。

业务链路：品牌方（SKG）花钱在流量平台（抖音、小红书等）投广告 → 把用户引到电商店铺（天猫 / 京东 / 抖音小店）→ 产生下单成交。投放的核心诉求只有一个：**花出去的广告费，换回来多少成交（值不值）**。

本系统就是把"花了多少、带来多少"这些数据，从各个投放平台**自动抓回来 → 统一口径 → 汇总成看板**，方便投手和管理层看效果、算账、做加投/关停决策。

> 行业黑话：投流 / 买量 / 效果广告；干这活的岗位叫 投手 / 优化师。

---

## 二、SKG 是谁

**SKG 是品牌方（广告主），做健康电器**——颈椎按摩仪、按摩椅那一类。属于**品牌电商**：自己在天猫、京东开旗舰店/自营店（数据里能看到"SKG天猫旗舰店""SKG-京东自营"），靠投广告拉动店铺销量。

它**不是电商平台，而是"在电商平台上卖货的品牌"**。我们这套系统服务的就是 SKG 的广告投放数据管理。

---

## 三、四个投放平台分别是干嘛的

**关键认知：这四个不是流量媒体，而是"第三方投放管理 / 服务商工具"。** 真正的媒体是抖音（巨量引擎）、小红书（聚光）。SKG 的广告由不同团队/代理，用不同工具在投，所以要把 4 套后台数据都抓回来汇总。

| 平台 | 本质 | 对接的媒体 / 成交 | 报表维度 |
|---|---|---|---|
| **小飞机** (smallfighter) | 巨量引擎买量工具 | 抖音/头条（巨量引擎），电商千川(CID) | 推广账号 / 广告组 / 广告 |
| **沸点** (fifay) | 抖音生态投放平台 | 抖音（含视频播放、店铺订单） | 账户 / 项目 / 计划 / 素材 |
| **微橙** (douyongtuan) | 巨量引擎投放平台 | 抖音投放，成交回流淘系(天猫/淘宝) | 账户 / 计划 / 素材 |
| **麦斯** (maxengine) | 小红书投放平台 | 小红书「聚光」广告 | 账户 / 计划 / 单元 / 创意 |

各平台后台字段五花八门，本系统已**统一归一为一套指标**入库（见下一节）。

## 四、数据指标怎么读（核心）

按"投放漏斗"从上到下理解最直观：花钱 → 曝光点击 → 转化成交 → ROI。

### ① 花钱
- **消耗 (cost)**：花出去的广告费（元）。最核心的成本项。

### ② 曝光 → 点击（广告触达）
- **展示量 (impressions)**：广告被展示了多少次
- **点击量 (clicks)**：被点了多少次
- **点击率 CTR = 点击 / 展示**：广告吸不吸引人
- **CPM**：每千次展示的成本；**CPC**：每次点击的成本

### ③ 转化 → 成交（钱花得值不值）
- **转化数 (conversions)**：完成目标动作的数量（如加购 / 表单 / 下单）
- **转化成本 = 消耗 / 转化数**：拉一个转化花多少钱，越低越好
- **订单数 (orders) / 付款金额 (pay_amount)**：广告带来的下单量、成交金额(GMV)

### ④ ROI —— 最关键的 KPI
- **ROI = 付款金额 / 消耗**，即"投产比 / 回报率"。
  - 例：花 100 元广告费，带来 300 元成交，**ROI = 3**。
  - **ROI > 1** 一般代表广告带来的销售额超过了广告花费（是否真赚钱还要看毛利，但投放侧主要盯 ROI）。**越高越好。**

### ⑤ "真实" 系列（real_*）
- **真实付款 / 真实订单 / 真实ROI**：去水分后的口径（剔除刷单、异常单，或更严格归因确认的真正成交），比普通口径更保守可信。
- **退款率 (refund_rate)**：成交后退款的比例，越高说明水分越大。

### ⑥ "直投归因" 系列（direct_*）—— 见第五节专项详解

## 五、直投归因（direct_*）专项详解

### 5.1 什么是"直投归因"

一条广告带来的成交分两种：

- **全部归因**（普通 orders / pay_amount / roi）：把该广告账户带来的**所有**成交都算上——包括用户看了广告过几天才回来买的（助攻/间接），或被引到店里却买了**别的**商品的。口径宽，容易"沾光"。
- **直投归因**（direct_*）：只算**这条广告 / 这个主推商品自己直接带来**的成交——用户点了这条广告、当场买了它主推的那个商品。口径严，是广告"自己的真实功劳"。

> 一句话：`direct_*` = 去掉蹭全店流量、去掉助攻后，这条广告自己实打实的战绩。

### 5.2 六个字段逐个解释

| 字段 | 含义 | 小飞机 | 沸点 | 微橙 | 麦斯 |
|---|---|---|---|---|---|
| `direct_orders` | 直投**下单量** | 直推付款订单数 | 直接原始订单量 | 单品订单数 | 主投品下单量 |
| `direct_pay_amount` | 直投**下单金额**(元) | 直推付款金额 | 直接原始订单金额 | 单品成交总金额 | 主投品下单金额 |
| `direct_roi` | 直投**下单ROI** | 直推订单roi | 直接原始Roi | 单品ROI | 主投品下单ROI |
| `direct_real_orders` | 直投**成交量** | 真实直推付款订单数 | 直接有效下单量 | 单品下单订单数 | 主投品成交量 |
| `direct_real_pay_amount` | 直投**成交金额**(元) | 真实直推付款金额 | 直接有效付款金额 | 单品下单总金额 | 主投品成交金额 |
| `direct_real_roi` | 直投**成交ROI** | 直推订单真实roi | 直接有效Roi | 单品下单ROI | 主投品成交ROI |

> 各平台叫法不同（直推 / 直接 / 单品 / 主投品），指的是同一件事，系统统一命名为 `direct_*`。

### 5.3 两个口径：「下单」 vs 「成交」

这 6 个字段是「3 个指标 × 2 个口径」：

- **下单口径**（`direct_orders` / `direct_pay_amount` / `direct_roi`）：用户**提交订单**就算，含未付款、后续退款的。反映**即时拉动力**。
- **成交口径**（`direct_real_orders` / `direct_real_pay_amount` / `direct_real_roi`）：**真实付款成交**才算，去掉水分。反映**真实到账收益**，更可信。

同一条广告，成交口径通常 ≤ 下单口径；两者差距大 = 下单多但真付款的少（水分/退款多），要警惕。

### 5.4 作用（为什么专门抓这组）

1. **判断单个广告/商品到底赚不赚钱**——只看全店 ROI 会被其它商品和自然流量抬高，`direct_*` 剔除干扰看广告自身效率。
2. **加投 / 关停决策**——投手据此决定某条广告该加预算还是砍掉：直投 ROI 好就加，差就停。
3. **识别"蹭量"广告**——对比 `direct_roi`（自己的功劳）和普通 `roi`（含全店/助攻）：接近说明确实能打；普通 ROI 高但直投 ROI 很低，说明是靠蹭全店流量"虚胖"。
4. **下单 vs 成交 双口径防水分**——发现刷单、高退款等异常。

### 5.5 举例

某条广告某天：消耗 100 元，普通 `roi` = 5（全店带来 500 元），但 `direct_roi` = 1.2（自己主推品只直接带来 120 元）。

> 解读：这条广告表面很好（ROI 5），但大部分成交是店里**别的商品**或助攻带来的，它主推的商品其实只勉强打平。若目标是推这个主推品，就别被虚高的全店 ROI 迷惑，得优化素材/定向。

## 六、一句话闭环

> SKG（品牌）在抖音/小红书投广告 → 用户看到(展示)、点击、下单(转化/订单) → 在天猫/京东/抖音店成交(付款金额) → 用花的钱(消耗)和换回的成交算出 **ROI**（含全店 ROI 与更严格的直投 ROI）→ 这些数由小飞机/沸点/微橙/麦斯四个工具各自记录 → **本系统自动抓回、统一口径、做成看板与导出**，让投手和老板一眼看清每个账户/每天投得值不值。
